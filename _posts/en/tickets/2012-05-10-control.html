Sold tickets

To prepare a list of sold tickets, select the performances you are interested in the Event Details page and click Sold Tickets.

The resulting page will display the selected performances and a list of sold tickets. This table can be sorted by clicking on the header cells. You may print this page (Ctrl + P on Windows, Cmd + P on Mac) and take it to the entrance to your event.

Security features

All tickets include the following items to validate at the access control point:

  1. The ticket type: Indicates the type that was purchased. You should confirm it using your printed list or Koliseo Checkpoint Charlie.
  2. The checksum: This field is automatically generated by the platform, and it should be the same that you have in your sold tickets list. Failure to validate this checksum field indicates a forged ticket.
  3. The QR code: contains the checksum value printed in a format that your android device can scan and validate automatically.

Tickets do not have to be printed to be valid. You can check in tickets from a cell phone screen all the same, just by using the security features described here.

The number of people indicated in the ticket should appear at the entrance to the event at the same time with the ticket.

Access control checklist

The following is the non-exhaustive list of things that you should check at the entrance.

  • Use android: You should try Koliseo Checkpoint Charlie from the Android Market. It makes the ticket validation process much faster, which helps to shorten your queues and make everything happen faster.
  • Always have a backup: Even if you are using Android, you should have a list of sold tickets printed in case your battery dies, the android device fails, or whatever. You should not rely exclusively on electronics, specially for big events.
  • Big events, split your queues: In case you are doing it by hand, you should organize your access control by user name (A-M, L-O, etc) to avoid having one single huge queue to resolve.
  • Internet Access: You don't need WiFi or 3G while validating tickets with Koliseo Checkpoint Charlie, but that means that you should synchronize your tickets before the event. This is done to prevent Internet access from collapsing during the first minutes of your event (likely to happen in big events).

Ticket alerts

Event organizers will be notified when the number of tickets available falls below a configurable threshold (by default 100,75,50,25,0). When any of these values is reached (for example with your first ticket sale, or when a performance gets sold out) the organizer will receive a automated notification message.

This list can be configured from your Edit user profile page. To turn alerts off, just leave the field empty.
